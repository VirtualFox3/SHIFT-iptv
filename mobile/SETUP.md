# SHIFT Mobile — Android / Fire Stick / Android TV

React Native (Expo) app using VLC for playback (supports MKV, HEVC, H.264, HLS — everything).

## Requirements

- Node.js 18+
- An [Expo](https://expo.dev) account (free) for EAS Build
- For Fire Stick / Android TV: ADB installed

No Android Studio or local Android SDK needed — builds happen in the cloud via EAS Build.

## First-time setup

```bash
cd mobile
npm install
npm install -g eas-cli
eas login
```

## Run in dev (Metro + Expo Go)

Expo Go can't load `react-native-vlc-media-player` (it's a native module), so use a
custom dev client build instead:

```bash
eas build --platform android --profile development
# install the resulting APK on your device/Fire Stick, then:
npm start
```

## Build an installable APK (no Play Store)

```bash
npm run build:preview
# EAS builds in the cloud and gives you a download link for the APK
# Sideload to Fire Stick: adb install shift-iptv.apk
```

## Build for Play Store (AAB)

```bash
npm run build:production
```

## Local native build (optional)

If you do have Android Studio/SDK installed locally, you can generate the native
project and build directly:

```bash
npm run prebuild
npm run android
```

The `android/` folder is generated by `expo prebuild` from `app.json` + the config
plugins in `plugins/` — it's not committed, since EAS Build regenerates it from
scratch on every cloud build.

## Features

- Xtream Codes login (same credentials as the website)
- M3U playlist support
- VLC player: plays MKV, HEVC, H.264, HLS — no "can't play" errors
- Continue Watching — synced separately from the website (localStorage vs device storage)
- Works on: Android phones, Android tablets, Fire Stick, Android TV boxes
- Same launcher icon shows in both the phone app drawer AND the Fire TV home screen
